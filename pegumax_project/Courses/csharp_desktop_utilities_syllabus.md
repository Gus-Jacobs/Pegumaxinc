# C# Desktop Utilities & Tool Automation

*A Windows developer's course on building lightweight, fast, native desktop tools with C#. Skip the bloat—create responsive utilities that handle files, automate the system, and feel instant to use.*

---

## Module 1: C# & .NET for Native Tools
**Set up a modern C# toolchain and understand the runtime so your utilities start fast and stay small.**

- The .NET runtime, project types, and choosing the right UI framework (WPF/WinForms/WinUI)
- Project setup, NuGet, and structuring a small, focused utility solution
- Build configurations and producing a lean, native-feeling executable

---

## Module 2: Object-Oriented Design in C#
**Apply solid OO design so your tools stay clean and extensible as they grow in scope.**

- Classes, interfaces, inheritance, and composition for maintainable structure
- Encapsulation, properties, and access modifiers for safe, clear APIs
- SOLID principles applied pragmatically to small utility codebases

---

## Module 3: Language Power Features
**Leverage C#'s modern features so you write less code that does more, reliably.**

- Generics, collections, and LINQ for expressive data manipulation
- Delegates, events, and lambdas for flexible, decoupled logic
- Nullability, pattern matching, and records for safer, cleaner code

---

## Module 4: Local File Handling
**Read, write, and manage files robustly so your utilities handle real-world data without choking.**

- The `System.IO` essentials: files, directories, paths, and streams
- Reading and writing text, binary, and structured formats (JSON/CSV)
- Watching the filesystem with `FileSystemWatcher` and handling errors safely

---

## Module 5: Asynchronous Loops & Responsiveness
**Use async/await so your tools process large workloads without ever freezing the UI.**

- `async`/`await` and the `Task` model for non-blocking operations
- Processing large files and batches asynchronously with progress reporting
- Cancellation tokens, throttling, and keeping long-running loops responsive

---

## Module 6: Building the UI
**Construct a clean, fast interface so your utility is as pleasant to use as it is functional.**

- Layout, controls, and data binding for a responsive desktop UI
- The MVVM pattern (or code-behind for simple tools) and when each fits
- Keeping the UI thread free and marshaling work back to it correctly

---

## Module 7: Custom Desktop Viewer Components
**Build reusable custom controls so your tools can display data exactly the way you need.**

- Creating custom and user controls for specialized display needs
- Virtualized lists and grids for rendering large datasets smoothly
- Custom rendering, styling, and packaging components for reuse across tools

---

## Module 8: System Automation & OS Integration
**Tap into Windows itself so your utilities automate tasks a manual workflow never could.**

- Launching, monitoring, and controlling processes from your tool
- Interacting with the registry, environment, and system settings safely
- Scheduling tasks, file-system automation, and reacting to system events

---

## Module 9: Configuration, Logging & Reliability
**Make your tools dependable and debuggable so they hold up under daily real-world use.**

- Persisting settings and user preferences across sessions
- Structured logging and diagnostics for troubleshooting in the field
- Exception handling strategy and failing gracefully without data loss

---

## Module 10: Packaging, Distribution & Performance
**Ship a polished, fast executable so users can install and trust your tool.**

- Publishing options: self-contained vs. framework-dependent and single-file builds
- Performance tuning: startup time, memory footprint, and allocation discipline
- Installers, code signing, and a simple update and distribution workflow

---
